# AI 幻觉

## 定义

**一句话：** AI 幻觉是指模型**自信地输出看似合理、语法通顺，但与事实不符或纯属编造**的内容——它优化的是「像人话、像连贯」，而非「句句可验证」。

**比喻：** 像一位**从不查资料、却总能把故事讲圆**的讲解员：听众觉得像真的，但细节可能是编的；模型训练目标是「预测下一个最可能的词」，不是「核查每个事实」。

> **对应关系：**「讲解员」对应**生成模型**；「不查资料」对应**无实时外部验证机制**；「故事讲圆」对应**流畅性与连贯性优化**；「编的」对应**幻觉内容**。

## 核心要点

- **成因机制**：
  - 训练目标为「预测下一 token」，追求**统计似然性**而非**事实准确性**
  - 知识有缺口、过时或冲突时，模型仍会**自信补全**而非标注不确定
  - 上下文不足或噪声时，模型可能**「脑补」**细节填补空白

- **常见表现**：
  - 编造不存在的论文、法条、链接、版本号
  - 把「可能」说成「一定」，把「部分」说成「全部」
  - 在用户给定材料外**捏造细节**（如「据文档 X 说…」实则文档无此句）
  - 生成**看似合理但无法运行**的代码（API 不存在、语法错误）

- **与口误/错误的区别**：
  - 普通错误：笔误、计算错，通常可识别
  - 幻觉：**虚构或过度断言**，语气高度自信，**难以从表面分辨**，需人或工具核验

## 在 Cursor/Claude Code 中的体现

| 场景 | 幻觉风险 | 产品对策 |
|------|----------|----------|
| **代码生成** | 编造不存在的方法、过时的 API、无法运行的代码 | 工具调用验证（运行测试、lint）、差异查看（Diff）确认 |
| **代码库理解** | 声称「某文件包含 X 函数」实则不存在 | 整个代码库索引、多文件交叉验证 |
| **工具使用** | MCP 调用时参数错误、返回结果误读 | 显式工具输出展示、人工确认敏感操作 |
| **长对话** | 多轮后「遗忘」早期约束、前后矛盾 | 自动记忆（Auto-memory）、`CLAUDE.md` 持久化规则 |

## 应用场景

- **评估输出**：默认预设「可能幻觉」，对关键事实（配置值、版本号、API 签名）要求**可验证来源**。
- **设计 Prompt**：要求「引用原文」「不确定时标注」「给出可运行的测试」。
- **选择工具**：用 RAG 接地、MCP 实时查询、人工审核，而非直接采信长文本生成。
- **阅读代码 Diff**：Claude Code 与 Cursor 均强调**差异审查**——让用户看见具体改了什么，降低「 silently 引入幻觉代码」的风险。
