# AI Agent

## 定义

**一句话：** AI Agent 是能**自主规划、使用工具、多轮执行**以达成目标的 AI 实体——它不仅回答问题，还能「动手做事」，在行动中观察反馈并动态调整。

**比喻：** 像一位**有明确 KPI 的项目经理**：接到目标后，自己会拆解任务、调度资源（工具/人）、分阶段执行，并根据中间结果调整计划，而不是一次性给出所有答案。

> **对应关系：**「项目经理」对应 **Agent 本身**；「KPI」对应 **用户给的目标/任务**；「拆解任务」对应 **规划（Planning）**；「调度资源」对应 **工具使用（Tool Use / MCP）**；「分阶段执行」对应 **行动-观察-调整的循环**；「调整计划」对应 **动态重规划（Re-planning）**。不较真之处：真实 Agent 的「规划」可能是隐式的（端到端训练）或显式的（思维链），但「目标驱动、多步执行、反馈调整」的直觉成立。

## 核心要点

- **与单次 LLM 对话的本质区别**：
  - LLM：你问一句，它答一句，**无状态、无后续**
  - Agent：有**记忆（上下文累积）**、有**循环（观察→思考→行动→再观察）**，能自主决定「下一步做什么」

- **关键组件**：
  - **规划（Planning）**：把大目标拆成可执行的子任务（「先查文档，再写代码，再测试」）
  - **工具使用（Tool Use）**：调用 MCP、API、代码执行器等改变外部世界
  - **记忆（Memory）**：短期（本轮上下文）+ 长期（跨会话的经验/知识）
  - **执行与反馈**：行动后观察结果，错误时重试或调整策略

- **与 Skill、MCP 的分工**：
  - **Skill**：告诉 Agent「怎么做」（工作手册、SOP）
  - **MCP**：给 Agent「能用什么工具」（外部接口）
  - **Agent**：「执行者」——理解目标，选择 Skill 和工具，动手完成

- **典型形态**：
  - **ReAct 模式**：思考（Reasoning）→ 行动（Acting）→ 观察（Observing）→ 再思考…
  - **Multi-Agent**：多个 Agent 分工（如「产品经理 Agent」提需求，「程序员 Agent」写代码，「测试 Agent」验 bug）

## 应用场景

- **自动化工作流**：「把需求文档转成代码、写测试、部署、发 PR」——Agent 自主拆解并执行多步
- **复杂信息收集**：「调研竞品功能，整理对比表」——Agent 循环搜索、归纳、验证
- **代码调试**：「这个报错什么原因」——Agent 读代码、查日志、改配置、运行验证，直到解决
- **长期任务管理**：「下周提醒我根据本周 bug 数据生成报告」——Agent 记 todo，到时触发执行

## 相关概念

- **Planning（规划）**：Agent 的「大脑」，决定任务分解和策略
- **Tool Use / MCP**：Agent 的「手臂」，连接外部世界
- **Memory**：Agent 的「记事本」，跨轮次保持状态
- **RAG**：Agent 调用知识库时的检索能力

## 来源

- 业界通称 Agent = 能自主感知、规划、行动、使用工具的 AI 系统（学术界 Multi-Agent Systems 概念延伸）
- ReAct、Reflexion、Plan-and-Execute 等是常见 Agent 架构模式
- Claude Code、AutoGPT、Devin 等是典型产品形态
