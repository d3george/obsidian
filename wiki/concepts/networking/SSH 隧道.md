# SSH 隧道

## 定义

SSH 隧道（SSH Tunneling）通过 SSH 协议建立加密通道，在其中封装其他网络流量。常用于端口转发、绕过防火墙或加密未加密协议的通信。

## 核心要点

### 动态端口转发（SOCKS 代理）

```bash
ssh -f -D <本地端口> -N -p <远程端口> user@远程主机
```

| 参数 | 含义 |
|------|------|
| `-f` | 后台运行（fork 到后台） |
| `-D` | 在本地启动 SOCKS5 代理 |
| `-N` | 不执行远程命令，仅做端口转发 |
| `-p` | 指定远程 SSH 端口 |

### 常见用途

- **SOCKS5 代理**：`-D` 参数在本地创建代理端点，所有支持 SOCKS 的应用可路由到此端口
- **端口转发**：将本地端口映射到远程主机的某个端口
- **加密传输**：所有通过隧道的流量都被 SSH 加密

### 保持隧道活跃

```bash
# SSH 客户端心跳，防止空闲断开
ServerAliveInterval=30
```

### 验证隧道状态

```bash
lsof -i :1080  # 查看指定端口是否有进程监听
```

## 应用场景

- 通过跳板机访问内网服务
- 在不可信网络上加密浏览器流量
- 将远程代理暴露到本地（如本方案中的 FortiClient + SSH）
