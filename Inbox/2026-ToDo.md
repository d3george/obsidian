## 2026 年度三大主线

| 主线 | 年度要赢的一件事 |
|------|------------------|
| **主职与效率** | 每日 9:30–12:30 深度工作不被打断，主职交付坚持「需求→AI V1→精修」，交付体感提速可复盘。 |
| **职业增值与收入** | 面试核心原理能讲清一轮；Upwork 有首单评价；二选一：上线 1 个 MVP 获 1 位付费用户，或拿到高薪远程面试/Offer。 |
| **健康与家庭** | 财务切换为定投组合并卸载盯盘 App；建立可执行的健康节奏；完成一次 10 天左右的「数字游民」家庭旅行。 |

## 第一阶段：生产力与面试基石（第 1–3 个月）

**本阶段目标：** 深度工作环境固化，面试「核心原理」能讲清，财务与健康冷启动。

### 关键结果

- [ ] **环境与流程（多数已完成）**
    - [x] 物理手机锁箱：每日 9:30–12:30 锁机。
    - [x] 浏览器安装 StayFocusd & Unhook，工作时段封禁 B 站、抖音推荐。
    - [x] Cursor + `.cursorrules`，主职坚持「描述需求 → AI 生成 V1 → 手动精修」。
- [ ] **面试复习：核心原理**
    - [ ] 前端：Event Loop 六阶段、微任务 vs 宏任务；TypeScript Medium（条件类型、infer）；RSC 与 Vue3 响应式对比，手写一个带缓存的 Hook。
    - [ ] 后端：PG MVCC、B-tree vs GIN、会读 `EXPLAIN ANALYZE`；Node/Hono：libuv 线程池、Stream 背压、Hono Edge 限制与优势。
    - [ ] 用 AI 模拟面试：针对 Hono 中间件模型与 PG 事务隔离级别提问并改进回答。
- [ ] **财务与健康**
    - [ ] 卸载同花顺；存量基金转为 80% 沪深300 + 20% 标普500 QDII 定投。
    - [ ] 晨会期间站立 + 15 分钟比目鱼肌提踵，本周执行天数记录在周复盘。

**本阶段完成定义：** 锁机与工具无倒退；能对着提纲讲完一轮「核心原理」；定投组合已切换；提踵习惯本月至少执行 3 次/周。

## 第二阶段：多维资产与 IP 启动（第 4–6 个月）

**本阶段目标：** Upwork 破冰（至少 1 条评价），个人 IP 有可计数的产出，系统设计能讲 3 个案例。

### 关键结果

- [ ] **Upwork 破冰**
    - [ ] Profile 标题：`Senior TS Developer | AI Integration & Micro-SaaS Expert`；录制 1 分钟英文介绍视频。
    - [ ] 通过 Direct Contracts 邀请熟人完成约 50 美元订单并留评，激活 Job Success Score。
    - [ ] 投递节奏：每日 10:00 / 14:00 / 18:00 检查新帖，只投前 10 名申请内的项目。
- [ ] **个人 IP：MCP 与内容**
    - [ ] 开发并开源 2 个 MCP 服务器（如 PostgreSQL 自然语言审计类）。
    - [ ] 公众号：每周 1 篇《AI 提效周报》系列，本阶段累计 ≥8 篇。
- [ ] **系统设计**
    - [ ] 攻克 ByteByteGo 中 URL 短链、即时通讯、分布式锁 3 个案例，能白板讲清；用 AI 练习「百万 DAU 聊天架构 + PG 写入瓶颈」。

**本阶段完成定义：** Upwork 有 ≥1 条真实评价；2 个 MCP 已开源；公众号本阶段 ≥8 篇；3 个系统设计案例能讲出。

## 第三阶段：产品变现与职业突围（第 7–9 个月）

**本阶段目标：** 上线 1 个 MVP 并获取首位付费用户，和/或进入高薪远程岗位的面试/Offer 流程。

### 关键结果

- [ ] **独立产品：21 天 Ship**
    - [ ] 选定一个痛点（如青岛本地企业小工具或 AI 内容生成器），技术栈锁定 Next.js + Tailwind + Supabase + Hono，立项到部署 Vercel ≤21 天。
    - [ ] 接入 LemonSqueezy 或 Stripe，发布到 V2EX、Product Hunt，目标：至少 1 位付费用户。
- [ ] **高薪远程投递**
    - [ ] 渠道：电鸭、V2EX 远程、LinkedIn Remote；面试中强调 Ownership：监控（如 Sentry）、成本与回滚。

**本阶段完成定义：** 至少达成其一：MVP 上线且有 1 位付费用户；或远程岗进入终面/Offer。

## 第四阶段：系统自动化与成果验收（第 10–12 个月）

**本阶段目标：** 内容与运营可复用、省时间；健康与生活升级；数字游民旅行成行。

### 关键结果

- [ ] **生产力与合规**
    - [ ] 用 AI 建立「技术文章→视频脚本」流程，IP 运营时间控制在每周约 2 小时内。
    - [ ] 若有第二份远程工作：独立工作台与显示器，设备不混用。
- [ ] **健康与生活**
    - [ ] 健身：每周 2 次力量训练（如哑铃划船、箭步蹲），记录在周复盘。
    - [ ] 年度旅行：一次约 10 天的数字游民试行（如大理或东南亚），上午处理工作，下午陪家人。

**本阶段完成定义：** 内容流程跑通且周耗时可接受；健身周频达标；旅行已出行或已订好行程。
