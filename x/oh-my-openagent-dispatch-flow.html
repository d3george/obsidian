oh-my-openagent — Agent 调度架构

Sisyphus 编排器 · 波次并行调度 · 规划流水线 · 会话连续性 · Ralph 循环

Sisyphus：编排规则

  • • 默认偏向委托 — 绝不单独工作
  • • 意图识别门每条消息强制执行（轮次局部重置）
  • • 通过依赖图进行波次并行执行
  • • 实现前必须通过上下文完整性门
  • • 遇到不良设计先提出挑战，给出替代方案
  • • 无开场白 — 直接开始工作

Agent 调度原则

  • • explore / librarian：始终后台 + 并行（10+ 个）
  • • oracle：只读，昂贵，结果出来前阻塞实现
  • • Prometheus：非平凡任务（2+ 步）通过 /start-work
  • • 反重复：绝不重复已委托的搜索
  • • task_id：后续跟进必须复用（节省 70%+ token）
  • • Sisyphus-Junior：不可再委托 → 防止无限循环

后台循环与验证

  • • 后台 Agent：派发 → 等待 system-reminder 通知
  • • 收到通知前绝不轮询 background_output
  • • 每次文件修改后运行 lsp_diagnostics
  • • 构建必须退出码为 0 — 手动 QA 不可省略
  • • Ralph Loop (/ulw-loop)：循环直到 100% 完成
  • • 绝不删除失败测试来让构建通过
