⌘输入与上下文

会话入口

Prompt
你发给 OpenCode 的直接任务描述，可包含文件、图片、约束和目标。
@ 文件
用 @path/to/file 明确拉入特定文件内容，降低误解成本。
图片输入
可拖入终端，为设计、界面或参考图提供额外上下文。

上下文

Context
当前会话可见的全部信息：对话历史、文件、规则、工具输出。
Compaction
上下文过长时自动压缩历史，保留关键摘要。
Summary
系统隐藏代理生成会话摘要，帮助长会话续航。
心法：Prompt 决定目标，Context 决定代理“看得到什么”。
↺会话与结果

常用能力

/undo
撤销代理在当前会话中的最近改动。
/redo
恢复刚撤销的改动。
/share
把当前会话生成可分享链接，默认不是自动公开。

补充概念

Snapshot
用于跟踪文件改动，支持回滚；可在配置中关闭。
Share
可配置为 manual / auto / disabled。
⚡规则与命令

规则文件

AGENTS.md
项目级规则中心。定义结构、验证顺序、惯例和注意事项。
全局 AGENTS
~/.config/opencode/AGENTS.md，用于个人全局偏好。
兼容 CLAUDE.md
若没有 AGENTS.md，OpenCode 可回退读取 Claude Code 规则。

自定义命令

目录
.opencode/commands/*.md，文件名即命令名。
用途
把重复工作流固化成 /ingest、/lint 等命令。
能力
支持参数占位、文件引用、shell 输出注入、指定 agent / model。
规则是长期约束，命令是可复用工作流。
⚙配置与目录

配置入口

opencode.json
项目级 JSON/JSONC 配置，可定义 model、agent、command、permission、mcp、plugin。
全局配置
~/.config/opencode/opencode.json。
TUI 配置
tui.json 管主题、键位、界面行为。

目录约定

.opencode
项目级目录，可放 commands/、agents/、plugins/、skills/。
instructions
在 opencode.json 里额外加载规则文件或远程规则。
🧠代理模式与协作

主代理

Build
默认主代理，适合真实实现与改动，通常有完整工具权限。
Plan
受限主代理，偏分析、方案、评审，默认对 edit/bash 更保守。
切换方式
在 TUI 中用 Tab 在主代理之间切换。

子代理

General
通用子代理，适合复杂研究、多步骤任务与并行工作。
Explore
只读子代理，适合找文件、搜索代码、解释结构。
@ 调用
可手动写 @general 或 @explore 调用。

自定义代理

目录
.opencode/agents/*.md。
可配置项
description、mode、prompt、model、permission、steps、color。
↳何时用哪个
Plan
需求不清、要评审、要先拆方案。
Build
目标明确，直接实现、修复、重构。
Explore
先摸底代码库，不想误改文件。
General
任务大、步骤多、适合拆出去独立完成。
🔧扩展、工具与权限

扩展能力

Skills
按任务触发的专业工作流，适合调试、设计、验证、图表等场景。
MCP Servers
把外部系统或工具接进来，例如数据库、浏览器、第三方服务。
Plugins
扩展 OpenCode 的工具、集成和行为。

内建工具

read / glob / grep
读取文件、搜文件、搜文本。
edit / write
修改与写入文件。
bash
执行命令，常用于测试、构建、Git、服务启动。
task
启动子代理，拆分复杂任务。
webfetch
拉取网页内容做研究或文档查询。

权限模型

allow
直接允许执行。
ask
执行前向用户确认。
deny
禁止使用对应工具或命令。
🚀上手路径
1. /init
初始化项目规则，生成或改进 AGENTS.md。
2. 选模式
先 Plan 想清楚，或直接 Build 干活。
3. 复用命令
把高频流程沉淀进 .opencode/commands/。
4. 控权限
用 permission 精细化控制 edit、bash、task。
5. 接外部能力
按需加 Skills、MCP、Plugins。
