Slash-Space 组织架构与运转

LLM Wiki 模式：人类提供来源与方向，AI Agent 自动维护知识层

Ingest 流程

  • • 人类将来源（文章、视频、文档）放入 raw/
  • • Agent 读取原始来源，理解内容
  • • 在 wiki/sources/ 创建来源摘要页
  • • 识别核心概念 → 创建或更新 wiki/concepts/
  • • 更新 wiki/index.md 导航与 wiki/log.md 日志
  • • 一个来源通常触发 10-15 个页面更新

Query 流程

  • • 人类向 Agent 提出问题
  • • Agent 先读 wiki/index.md 定位相关页面
  • • 深入读取概念页与来源页，必要时回溯 raw/
  • • 基于 wiki 已有知识综合回答，附 [[引用]]
  • • 有价值时沉淀为 comparisons/ 或 synthesis/
  • • 追加操作记录到 wiki/log.md

分工与约束

  • • 人类：提供信息来源、确定方向、审阅质量
  • • Agent：自动维护 wiki，保持知识最新与一致
  • • AGENTS.md：定义项目结构、约定、文风
  • • raw/ 不可变：Agent 只读，人类写入
  • • wiki/ Agent 拥有：自动写入、交叉引用、更新
  • • Lint 定期检查：孤立页、死链接、矛盾、缺口
