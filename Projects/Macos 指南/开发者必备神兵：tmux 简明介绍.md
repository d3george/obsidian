这是一篇关于 tmux 的简短介绍以及常用命令列表。

---

在远程服务器上工作时，你是否遇到过因网络断开而导致 SSH 连接中断，正在运行的耗时脚本也随之终止的尴尬情况？或者，你是否厌倦了在多个终端窗口之间疯狂切换，难以管理？

`tmux` 就是为了解决这些痛点而生的。

`tmux` 是 Terminal Multiplexer（终端复用器）的简称。简单来说，它就像是终端里的“窗口管理器”和“持久运行器”。

#### 为什么你需要 tmux？

1. **会话持久化 (Persistence):** 这是它最强大的功能。通过 tmux 启动的程序，即使你断开了 SSH 连接，它们依然会在后台继续运行。当你再次连接服务器时，可以随时“重连”（attach）到之前的会话，一切如初。
    
2. **灵活的布局 (Layouts):** tmux 允许你在一个终端窗口中同时水平或垂直分割出多个更小的终端区域（称为“面板”）。你可以一边编辑代码，一边查看日志，一边运行测试，极大地提升效率。
    
3. **高效的组织 (Organization):** 你可以在一个会话中创建多个独立的“窗口”（就像浏览器里的标签页），每个窗口专注于不同的任务。
    

#### tmux 的核心概念

`tmux` 的结构清晰，分为三个核心层级：

- **会话 (Session):** 一个持久的工作空间，通常代表一个特定的项目或任务。
    
- **窗口 (Window):** 会话中的全屏区域，像浏览器标签页，一个窗口可包含多个面板。
    
- **面板 (Pane):** 窗口内的分割区域，是最终运行终端命令的地方。

### 常用 tmux 命令列表

#### 1. 会话管理 (在普通终端中运行)

|**功能**|**命令**|
|---|---|
|新建一个会话|`tmux`|
|新建并命名会话|`tmux new -s <会话名称>`|
|列出所有会话|`tmux ls`|
|连接回上一个会话|`tmux attach`|
|连接到指定会话|`tmux attach -t <会话名称>`|
|彻底关闭指定会话|`tmux kill-session -t <会话名称>`|

---

#### 2. 快捷操作 (在 tmux 会话中运行)

在 tmux 中进行操作，几乎都要先按一个“前缀键”（Prefix Key），默认是 **`Ctrl + b`**。

你无需一直按住它们，只需按一下 `Ctrl + b`，松开，然后再迅速按具体的功能键。

**通用操作**

|**功能**|**快捷键**|
|---|---|
|分离当前会话（回到普通终端，后台运行）|`Ctrl+b`, `d`|
|进入命令模式（类似 Vim 的 `:` 模式）|`Ctrl+b`, `:`|
|显示所有快捷键帮助|`Ctrl+b`, `?`|

**面板 (Pane) 管理**

|**功能**|**快捷键**|
|---|---|
|垂直分割面板|`Ctrl+b`, `%`|
|水平分割面板|`Ctrl+b`, `"`|
|导航（切换）到不同面板|`Ctrl+b`, `箭头键`|
|关闭当前面板|`Ctrl+b`, `x` (或输入 `exit`)|
|调整当前面板大小|按住 `Ctrl+b`，按 `箭头键`|
|面板最大化/还原 (Toggle zoom)|`Ctrl+b`, `z`|

**窗口 (Window) 管理**

|**功能**|**快捷键**|
|---|---|
|新建一个窗口|`Ctrl+b`, `c`|
|切换到下一个窗口|`Ctrl+b`, `n`|
|切换到上一个窗口|`Ctrl+b`, `p`|
|切换到指定编号窗口 (0-9)|`Ctrl+b`, `<数字>`|
|重命名当前窗口|`Ctrl+b`, `,`|
|列出所有窗口（进行可视选择）|`Ctrl+b`, `w`|

### Session 持久化

默认情况下，`tmux` 的所有 session 都是存储在内存中的，一旦系统重启或 `tmux` 进程被杀死，这些信息确实会消失。

要实现“永不丢失”的 session，目前最成熟、最通用的方案是使用 `tmux-resurrect` 插件，配合 `tmux-continuum` 实现自动化保存。

---


#### 1. 安装 TPM (Tmux Plugin Manager)
如果你还没有安装 `tmux` 插件管理器，建议先安装它，这是管理插件最简单的方式。

在终端执行：
```bash
git clone https://github.com/tmux-plugins/tpm ~/.tmux/plugins/tpm
```

#### 2. 配置 `.tmux.conf`
编辑你的 `~/.tmux.conf` 文件（如果没有则新建），添加以下内容：

```bash
# 启用 TPM
set -g @plugin 'tmux-plugins/tpm'
set -g @plugin 'tmux-plugins/tmux-sensible'

# --- 关键插件 ---
# 1. 保存和恢复 session 的基础插件
set -g @plugin 'tmux-plugins/tmux-resurrect'
# 2. 自动保存并自动启动（依赖 resurrect）
set -g @plugin 'tmux-plugins/tmux-continuum'

# --- 插件配置 ---
# 开启自动恢复 session
set -g @continuum-restore 'on'
# 设置自动保存间隔（单位：分钟）
set -g @continuum-save-interval '15'
# (可选) 恢复面板内容（实验性）
set -g @resurrect-capture-pane-contents 'on'

# 初始化 TPM (这行必须放在最后)
run '~/.tmux/plugins/tpm/tpm'
```

#### 3. 加载配置并安装
1. 进入 `tmux`。
2. 按下 `Prefix` + `I`（默认前缀键一般是 `Ctrl + b`，然后按大写的 `I`）。
3. 插件会自动下载并安装。

---

#### 4. 🎮 如何使用

一旦安装好这些插件，你的 `tmux` 就具备了“记忆”能力：

##### 手动操作 (tmux-resurrect)
* **保存：** 按 `Prefix` + `Ctrl-s` (Save)。你会看到状态栏显示 "Saved"。
* **恢复：** 按 `Prefix` + `Ctrl-r` (Restore)。

##### 自动操作 (tmux-continuum)
* **自动保存：** 插件会每隔 15 分钟在后台悄悄保存一次，你不需要操作。
* **重启后自动恢复：** 当你关机重启后，只要你再次打开 `tmux`，它会自动检测到保存的快照并把所有窗口、面板、路径恢复如初。

---


总结来说，如果你经常需要在终端里进行多任务处理，或者需要在远程服务器上运行不稳定的脚本，`tmux` 绝对是你提高工作效率和确保工作连续性的终极利器。
