# AI 代理核心概念速查

根据 **Cursor** 和 **Claude Code** 的官方文档以及最新的 AI 开发实践，这些术语共同构成了一套完整的“AI 代理（Agent）操作系统”。

![](./assets/AI%20代理核心概念速查/image-2026-03-07-20-02-00.png)

以下是这些通用定义的深度解析：

### 1. 基础交互层：Prompt & Context

这是你与 AI 沟通的入口和背景。

* **Prompt (提示词)**：你发给 AI 的直接指令。在现代 AI 编辑器中，Prompt 不只是文字，还包括通过 `@` 符号挂载的特定文件、文档或代码片段。
* **Context (上下文)**：AI 在处理你的请求时“能看到”的所有信息。
  * **组成**：包括你的对话历史、打开的文件、项目索引（Index）以及特定的项目规则。
  * **管理**：Context 是极其昂贵的资源（Token 消耗）。Cursor 和 Claude Code 的核心竞争力就在于如何通过算法自动筛选出最相关的代码，放入这个“有限的窗口”里。

---

### 2. 能力扩展层：Skills & MCP

这是 AI 的“知识储备”与“外部工具箱”。

* **Skills (技能)**：**模块化的专业工作流。**
  * **定义**：它是针对特定任务（如：React 最佳实践、单元测试编写）封装的一套 Prompt 指令集。
  * **特点**：它是“按需加载”的。只有当任务匹配时，Skills 里的规则才会注入到 Context 中，从而节省空间并提高准确性。


* **MCP (Model Context Protocol)**：**AI 的通用接口协议。**
  * **定义**：由 Anthropic 发布的开放标准，被誉为“AI 界的 HTTP 协议”。
  * **作用**：它让 AI 能安全地连接到外部数据（如 Google Drive、Slack、数据库）或调用本地工具（如文件读写、Bash 脚本）。

 **区别**：***Skills 是 AI 脑子里的“经验”，而 MCP 是 AI 手里的“工具”***。

---

### 3. 执行协作层：Subagent & Worktree

这是 AI 处理复杂任务时的“分身”与“试验场”。

* **Subagent (子代理)**：**专项任务处理单元。**
	* **场景**：当你让 AI 重构整个项目时，主 Agent 会启动一个 Subagent 专门负责搜索文件，另一个负责修改，它们拥有独立的 Context 窗口。
	* **优势**：并行处理任务，且防止主对话框被大量的中间过程（如搜索结果）淹没。


* **Worktree (工作树)**：**环境隔离机制。**
	* **定义**：基于 Git 的概念。在 AI 自动执行高风险操作（如大规模重构）时，它会创建一个临时的 Worktree。
	* **作用**：让 AI 在一个完全隔离的临时目录里折腾。只有当它完成任务并运行测试通过后，才会让你合并到主分支，确保你的开发环境不会被 AI 搞乱。

---

### 4. 规则约束层：Rules (CLAUDE.md / .cursorrules)

这是 AI 在你项目里的“员工守则”。

* **Rules (规则)**：持久化的指令。
  * **Cursor** 使用 `.cursorrules` 或 `.cursor/rules/*.mdc` 文件。
  * **Claude Code** 使用 `CLAUDE.md` 文件。

* **作用**：规定项目的代码风格（如“始终使用 Tailwind”）、架构约定（如“禁止修改 legacy 文件夹”）等。**AI 在每次回答前都会先阅读这些守则**。

---

### 5. 执行与扩展机制：Commands、Plugins、Hooks

* **Commands（命令）**
  * **定义**：用户或系统预定义的「一键工作流」。在 Cursor 中对应 `.cursor/commands/*.md`，调用时 AI 按该文件中的步骤执行（如整理笔记、提取概念）。
  * **特点**：可复用、可分享；把固定流程固化进命令，减少重复说明。

* **Plugins（插件）**
  * **定义**：扩展 AI 或编辑器能力的模块。在 Cursor/IDE 中常指扩展功能（如 MCP 服务器、语言/格式化插件）。
  * **与 MCP 的关系**：MCP 是连接外部数据与工具的协议；Plugins 是实现该协议或其它扩展能力的具体载体（如某个 MCP server、某个 IDE 插件）。

* **Hooks（钩子）**
  * **定义**：在特定时机自动触发的逻辑（如「对话前」「工具调用后」）。
  * **作用**：在生命周期节点注入规则、校验或后续处理；常见于 CI（Git hooks）、前端（React Hooks），在 AI 代理中可对应「请求前/后」「子任务完成时」等钩子。

### 📊 核心概念速查表

| 术语 | 形象类比 | 核心价值 |
| --- | --- | --- |
| **Prompt** | 需求订单 | 明确目标 |
| **Context** | 工作记忆 | 决策依据 |
| **Skills** | 专项证书/经验 | 提升专业度 |
| **MCP** | 外部插件插座 | 连接真实世界数据 |
| **Subagent** | 外包小弟/分身 | 解决复杂、并行任务 |
| **Worktree** | 临时试验台 | 保证生产环境安全 |
| **Rules** | 员工守则 | 持久化约定 |
| **Commands** | 快捷指令/宏 | 固化工作流、一键执行 |
| **Plugins** | 扩展模块/插座实现 | 扩展能力与集成 |
| **Hooks** | 生命周期触发器 | 在关键节点注入逻辑 |
