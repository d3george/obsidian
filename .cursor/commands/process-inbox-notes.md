# process-inbox-notes

**定位**：对当前这篇 Inbox 笔记做一次结构化分析，并将分析块追加到文末（或替换已有分析块）。不负责分类、打标签、提取概念——那些由其他命令完成。

---

## 你的任务

1. 以**当前打开或 @ 的这篇笔记**为对象（通常来自 Inbox：摘录、剪藏或外部导出的初稿）。
2. 基于正文做分析；**不要复述或照抄原文**，只生成下面这一块内容。
3. 若文末已有「## AI 分析 (Processed by Cursor)」整段，则用本次结果**整体替换**该段；否则**追加**到文件末尾。
4. 分析内容全部使用**简体中文**。

## 必须遵守的输出格式

以下结构必须完整保留，仅替换方括号中的内容。以一条水平线 `---` 开始。

```
---

## AI 分析 (Processed by Cursor)

### 1. Summary (内容总结)
[2–4 句话概括全文核心论点或信息。]

### 2. Key Takeaway (内容要点)
- [要点一]
- [要点二]
- [要点三]
[按需 3–5 条，每条一句。]

### 3. Something New (值得关注的信息)
[一段话：指出文中新颖、易被忽视或与更广趋势/概念相关的点；可自然提及与库内已有概念的关联，便于后续加 [[链接]]。]
```

## 规则

- Summary 与 Key Takeaway 必须紧扣当前文档内容。
- Something New 可适度延伸，但需有据、可追溯。
- 若笔记几乎为空或仅有元数据，可依标题/来源做简短推断分析；有正文时一律基于正文。
