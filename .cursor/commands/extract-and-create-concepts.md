# extract-and-create-concepts

从一篇来源笔记（summary）中识别核心概念，为每个概念创建或更新原子化概念笔记，并在原笔记中添加链接。一次性完成「识别 → 创建/更新 → 链接」。

## 使用场景

- 有一篇摘要笔记（读书笔记、视频总结、文章概要），需要将其中的概念提炼成独立笔记
- 希望一次性完成概念提取与概念笔记的创建/更新，避免分步执行

## 工作流程

1. 分析当前笔记，识别其中的核心概念（只提取真正的概念，不提取过于具体的内容）
2. 对每个概念：若概念笔记已存在则更新并添加来源链接，若不存在则在 `Resources/Topics/[领域]/` 下创建新笔记
3. 在原笔记中添加指向所有概念笔记的链接（如「## 相关概念」小节）
4. 为概念笔记添加适当的 `type/concept`、`topic/` 标签及双向链接

## 概念笔记输出格式

在 `Resources/Topics/[领域]/[概念名].md` 创建或更新笔记，包含：

```markdown
---
tags:
  - type/concept
  - topic/[相关主题]
---

# [概念名]

## 定义
[用你自己的话描述这个概念]

## 核心要点
- [要点1]
- [要点2]

## 应用场景
[何时使用这个概念]

## 相关概念
- [[相关概念1]]
- [[相关概念2]]

## 来源
- [[当前来源笔记名]]
```

## 输出要求

- 列出所有识别到的概念
- 为每个概念创建或更新笔记，确保包含：清晰定义、核心要点、来源链接、适当标签
- 在原笔记末尾或合适位置添加「相关概念」列表并写入 `[[概念笔记名]]` 链接

## 注意事项

- 只提取真正的概念，不要提取过于具体的内容
- 如果概念笔记已存在，更新它并追加来源链接，不要创建重复笔记
- 保持概念的原子性（一个笔记一个概念）
- 若概念属于多个领域，选择最主要的领域作为目录
